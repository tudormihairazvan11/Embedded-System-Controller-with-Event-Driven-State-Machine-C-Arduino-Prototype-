# Embedded-System-Controller-with-Event-Driven-State-Machine-C-Arduino-Prototype-
-designed and implemented an embedded system controller using an event-driven state machine architecture. The firmware processes analog sensor input, handles user interaction via button events, and controls visual and acoustic outputs using non-blocking timing logic. The project was developed software-first in C and later ported to Arduino for hardware validation.  
Designed a finite state machine (FSM) with multiple operational states (IDLE, MONITORING, WARNING, ERROR)

Implemented event-based logic separating system inputs from decision-making

Integrated analog sensor acquisition and digital I/O control

Applied non-blocking timing techniques using timer-based logic instead of delays

Developed firmware using a modular, software-first approach, enabling easy portability to other microcontrollers


SKILLS:
-C(embedded)
-Arduino
-Analog sensor interffacing
-GPIO(INPUT/OUTPUT)
-State machine logic
-Modular firmware design
